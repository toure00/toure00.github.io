Magit

Magit 是 emacs 中的 git 工具，这里记录一下我研究 magit 的笔记 ，如有错误，请在留言区指出。

Magit 快捷键

快捷键 作用
SCP g i 初始化git仓库
SCP g s 进入magit status
SCP g f l 查看某一个文件的变化历史
C-x l 进入专家模式
l l 查看当前分支log历史
l o 查看其他分支历史
l a 查看所有分支历史
b b 切换commit(只移动HARD指针),光标下的commit为第一候选
b l 切换分支
b c 新建分支
b x 删除分支
b X / o 切换分支和HARD指针到当前光标下的commit(reset)
r i 交互式合并commit ri
r k 移除 commit
r m 修改 commit
r w 改写 commit summary
r s 合并分支(常用于master分支=>其他分支)
(不显示合并记录,log为一条直线,会移除当前分支的提交记录)
(不要在公共分支上使用 rebase)
m m 合并分支(常用于其他分支 => master)
(显示合并记录,log为多条分支线,能保存完整的提交记录)
c c 提交commit
c a 修改当前commit的summary(不生成新的提交)
c e 将当前暂存区的内容整合进当前commit(不产生新的提交)
p u push到远程仓库
p u push to upstram最普通的git push
p e push to elsewhere会提示你push到哪个远程分支
p o push another branch to会提示你push哪个分支
p T push a tagpush一个tag标签
p t push all tagpush所有tag标签
f u fetch上游仓库到本地(会覆盖工作区和暂存区的修改)
f p fetch远程仓库到本地
f e 提示你从哪fetch
F u pull上游仓库到本地(会覆盖工作区和暂存区的修改)
F p pull远程仓库到本地
F e 提示你从哪pull
h 查看magit帮助文档
h g 刷新
C u 克隆远程仓库(url)
M a 建立本地仓库与远程仓库分支的关联
s add 光标下的文件
S add 所有文件
vs add 选中的修改内容
u 撤销 add 光标下的文件
U 撤销 add 所有文件
vu 撤销 add 选中的修改内容
x 删除光标下新增的修改(文件和内容)
X 列出项目中的所有文件,可选择删除
y r 列出项目中的所有的本地/远程分支和tag 可用x删除分支
y b 复制commit的hash code
D 比较commit
d 比较不同commit
d r 比较选中范围内的commit(第一个和最后一个)
e 使用ediff比较不同commit的文件(手动选择要比较的文件)
比较选中范围内的commit(第一个和最后一个)中的文件
如果光标在Staged或Unstaged上,则比较三个阶段的文件差异
z p / A 找回 stash 的修改
z k

Rebase

可以实现调整commit 的顺序，合并commit,放弃某个commit等，下面这些按键绑定是使用evil-magit后的按键绑定，这些绑定不必担心记不住，用到的时候magit会以注释的形式展示给你

快捷键 作用
p pick = use commit 保留此commit
r reword = use commit, but edit the commit message 只修改此次commit 的日志
e edit = use commit, but stop for amending
s squash = use commit, but meld into previous commit 把这条commit与上一条commit合并(会提示输入合并之后的日志)
f fixup = like “squash”, but discard this commit’s log message 同squash,只是合并后的那条commit直接使用上条commit的日志
x exec = run command (the rest of the line) using shell
d drop = remove commit 丢弃此条commit
u undo last change
C-c C-c tell Git to make it happen 执行更改
C-c C-k tell Git that you changed your mind, i.e. abort 放弃更改
k move point to previous line
j move point to next line
M-k move the commit at point up
M-j move the commit at point down
RET show the commit at point in another buffer
